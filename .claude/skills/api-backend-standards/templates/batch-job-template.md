# バッチジョブ仕様テンプレート

## ジョブ名: [ジョブ名]

| 項目 | 内容 |
|------|------|
| ジョブ種別 | スコア計算 / PDF生成 / Instagram取得 / etc |
| トリガー | API呼び出し / スケジュール / イベント |
| 処理方式 | 非同期キュー |
| べき等性 | あり / なし |
| 最大リトライ数 | 3 |
| タイムアウト | [秒] |

## 処理フロー

```
1. ジョブ受付（jobId 発行）
2. キューに追加
3. ワーカーで処理
   3.1 [処理ステップ]
   3.2 [処理ステップ]
4. 結果を DB に保存
5. ステータスを completed / failed に更新
```

## べき等性設計

```
IF ジョブが既に completed THEN
  既存結果を返す（再実行しない）
ELSE IF ジョブが processing THEN
  処理中ステータスを返す
ELSE
  新規実行
```

## エラーハンドリング

| エラー種別 | 処理 |
|-----------|------|
| 一時的エラー（429 / 503） | exponential backoff でリトライ |
| 永続的エラー（404 / 422） | リトライなし、failed に更新 |
| タイムアウト | リトライ（最大 3 回） |
| 不明エラー | 1 回リトライ後 failed |

## レート制限考慮（Instagram API の場合）

- 同一ユーザーの Instagram API 呼び出しはシリアル実行（並列化しない）
- 呼び出し間隔: 最低 0.5 秒
- 429 エラー時: `Retry-After` ヘッダーを参照して待機

## ログ出力

```json
{
  "jobId": "...",
  "jobType": "...",
  "status": "started | completed | failed",
  "durationMs": 0,
  "errorCode": null
}
```

※ PII（username 等）はハッシュ化してログに出力する
