# 受け入れ条件テンプレート

## 形式（Given / When / Then）

```
AC-[番号]: [条件名]

Given: [前提条件（システムや環境の状態）]
When:  [ユーザーの操作・トリガー]
Then:  [期待される結果]
And:   [追加の期待結果（必要に応じて）]
```

## 例（このプロジェクト固有）

### Instagram プロフィール取得

```
AC-001: 公開アカウントの分析

Given: ユーザーが analyst ロールでログインしている
When:  公開 Instagram アカウントの URL を入力して分析を実行する
Then:  フォロワー数・投稿数・エンゲージメント率（推定）が表示される
And:   各データに「事実」「推定（信頼度）」のラベルが付いている
And:   LLM 生成コメントに「AI分析」のマークが付いている
```

```
AC-002: 非公開アカウントの分析

Given: ユーザーが analyst ロールでログインしている
When:  非公開 Instagram アカウントの URL を入力して分析を実行する
Then:  「このアカウントは非公開のため分析できません」が表示される
And:   部分的に取得可能な情報がある場合は表示し、不可のデータは「取得不可」と表示する
```

### スコア表示

```
AC-003: スコアの信頼度表示

Given: インフルエンサー分析が完了している
When:  スコア詳細を確認する
Then:  各サブスコアに信頼度（high/medium/low）が表示される
And:   信頼度が low のスコアには警告アイコンが表示される
And:   スコアの根拠が表示される
```

## 品質チェック

- [ ] テスト可能か（「分かりやすい」はテスト不可）
- [ ] 「事実/推定/LLM生成」の区別が明記されているか
- [ ] エラーケース・境界ケースが含まれているか
- [ ] 非機能要件（速度・セキュリティ）が必要なら含まれているか
