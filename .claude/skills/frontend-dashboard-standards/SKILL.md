# Frontend Dashboard Standards

## データ表示の原則（最重要）

事実/推定/LLM生成を必ず視覚的に区別する:

| データ種別 | 表示スタイル | ラベル |
|-----------|------------|--------|
| fact（事実） | 通常表示 | なし（または[事実]） |
| estimated（推定） | グレー系 + バッジ | [推定] + 信頼度アイコン |
| llm_generated | AI アイコン付き | [AI分析] |
| unavailable（取得不可） | ダッシュ (—) | [取得不可] ツールチップ |

信頼度インジケーター:
- high: グリーン dot
- medium: イエロー dot
- low: レッド dot + 警告アイコン

## 状態管理（全画面必須）

| 状態 | 表示 |
|------|------|
| ローディング | スケルトンローダー（Instagram API は遅い） |
| 空状態（データなし） | 「候補がありません。URLを入力して追加してください」 |
| エラー | エラーメッセージ + 再試行ボタン |
| 非公開アカウント | 「非公開アカウントのため分析できません」 |
| スコア計算中 | プログレスバー |

## テーブル設計

`templates/table-filter-template.md` を参照。

必須機能:
- ソート（列ヘッダークリック）
- フィルタ（フォロワー数・ジャンル・スコア範囲）
- ページネーション（1ページ 20 件）
- 列の表示/非表示切り替え

## 比較ビュー

- 最大 10 件の横並び比較
- 10 件超では横スクロール
- 選択した候補を固定ヘッダーに表示
- 数値は棒グラフで直感的に比較できるようにする
- 推定値と事実値で色分けする

## フォーム入力

Instagram URL / username の入力バリデーション:
```
有効な形式:
- https://www.instagram.com/username/
- https://instagram.com/username
- @username
- username

無効な形式:
- https://www.tiktok.com/... （Instagram ではない）
- 空文字
```

## レスポンシブ対応

- デスクトップ主体（1280px 以上で最適）
- タブレット（768px〜1279px）で動作
- モバイルは「制限あり表示」で対応（比較ビューは非推奨）

## gotchas

- Instagram API の応答は 2〜10 秒かかることがある。必ずスケルトンを表示する
- LLM 生成テキストを innerHTML に直接挿入しない（XSS 対策）
- 比較ビューで 10 件固定幅にすると小さい画面で見切れる（横スクロール必須）
