# コンテキスト管理ルール

## コンテキストを節約する原則

- 不要なファイル全文をペーストしない。必要な部分だけを抜粋する
- 大きな JSON レスポンスをそのまま会話に貼り込まない
- 繰り返し使う情報は skill / rules / docs に書き、参照する形にする

## 作業開始前に必ずやること

1. `repo-mapping` skill でリポジトリ構成を把握する
2. 関連する既存ファイルを読んでから変更を提案する
3. 実装方針は `architecture-decision-record` skill に記録する

## 長い作業の管理

- 複数ステップの作業は TodoWrite でタスク分割してから進める
- 大きなタスクは chief-orchestrator に委任して分割する
- 各ステップで「何をしたか」「何が残っているか」を短く記録する

## agent 間の情報受け渡し

- agent 間は成果物ファイル（仕様書・ADR・テスト結果）を通じて情報を渡す
- 会話内でのコンテキスト直接共有は最小化する
- 成果物は `docs/agent-system/` または適切な場所に保存する

## 探索の効率化

- ファイルを探すときは Glob を使う（`find` コマンドより速い）
- 内容を探すときは Grep を使う（`grep` コマンドより速い）
- 広範な探索が必要な場合は Agent(Explore) を使う

## gotchas

- 大きなファイルを全文読むと他の重要な情報が押し出される。必要箇所だけ読む
- 複数の agent を同時起動するとコンテキストが分散する。成果物ファイルで同期する
