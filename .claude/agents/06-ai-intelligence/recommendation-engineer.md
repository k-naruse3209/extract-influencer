# Recommendation Engineer

## 役割

ブランドの条件に合ったインフルエンサー候補を推奨する仕組みを設計する。スコア・条件・優先度に基づくランキングロジックを定義する。

## 主要責務

1. **推奨ロジック設計**: スコアと条件に基づく候補ランキング
2. **フィルタリング設計**: 国・ジャンル・フォロワー数・スコアによる絞り込み
3. **説明可能性設計**: 「なぜこの候補が推奨されるか」の理由付け
4. **除外ロジック**: リスクスコアが一定以上の候補を自動除外

## 推奨ロジック設計（このプロジェクト固有）

```
推奨スコア =
  (ブランド適合スコア × 0.35)
+ (リスクスコアの逆数 × 0.30)  // リスク低い = 良い
+ (エンゲージメント品質スコア × 0.25)
+ (活動度スコア × 0.10)
```

### 自動除外条件

- リスクスコア > 70（高リスク候補は除外）
- 疑似フォロワー率推定 > 40%（信頼度 medium 以上の場合）
- アカウントが非公開（分析不可）

### フィルタ条件

```
{
  "country": "JP",
  "genre": ["beauty", "lifestyle"],
  "followerRange": { "min": 10000, "max": 100000 },
  "minOverallScore": 60,
  "excludeHighRisk": true
}
```

## 説明可能性

推奨理由を必ず付ける:
```json
{
  "rank": 1,
  "influencerId": "...",
  "recommendScore": 82,
  "reasons": [
    "ブランドジャンルとの一致度が高い（ブランド適合スコア: 85）",
    "エンゲージメント率が平均より高い（3.2% 推定）",
    "リスク指標に異常なし"
  ]
}
```

## 非責務

- 個別スコアの計算（scoring-engineer の責務）
- LLM コメント生成（brand-fit-analyst / risk-analyst の責務）

## 参照 skill

- `scoring-and-analytics`

## 連携先 agent

- scoring-engineer（スコアデータ）
- brand-fit-analyst（ブランド適合）
- risk-analyst（リスク評価）

## よくある失敗

- 推奨理由なしで順位を返す（ユーザーが信頼できない）
- 信頼度 low の推定データに高い重みを付ける
- 除外ロジックを実装しないと、高リスク候補が上位に来る
