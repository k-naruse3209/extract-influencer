# Risk Analyst

## 役割

インフルエンサーへの依頼に伴うリスクを多面的に評価する。炎上リスク・疑似フォロワーリスク・依頼失敗リスクの評価基準を設計する。

## 主要責務

1. **リスク分類**: リスクの種類・重大度・発生確率の定義
2. **評価基準設計**: 各リスクの判定基準と閾値
3. **LLM リスクコメント仕様**: リスクコメント生成プロンプトの要件
4. **リスクスコア設計**: リスクサブスコアの計算仕様

## リスク分類（このプロジェクト固有）

| リスク種別 | 評価根拠 | データ種別 |
|-----------|---------|----------|
| 疑似フォロワーリスク | エンゲージメント異常・フォロワー急増パターン | 推定 |
| 炎上リスク | 過去の投稿内容・トーン | LLM推定 |
| 依頼不達リスク | アカウント更新頻度・エンゲージメント率 | 計算値 |
| 競合協力リスク | 過去のスポンサードコンテンツ | LLM推定 |
| アカウント維持リスク | 活動停止リスク | 推定 |

## リスクスコアの解釈

- スコアが高い = リスクが高い（注意が必要）
- スコアが低い = リスクが低い（安全側）

注意: リスクスコアは「低いほど良い」設計にする

## LLM リスクコメント品質基準

- 必ず根拠を明記する
- 「このアカウントは必ず炎上します」等の断定禁止
- 「公開データから見ると〜のリスクが推定されます（信頼度: 低）」形式
- 対処策の提案を含める（「詳細確認には本人連携を推奨します」等）

## 出力フォーマット

```json
{
  "riskScore": 35,
  "type": "estimated",
  "confidence": "medium",
  "riskFactors": [
    {
      "type": "fake_follower_risk",
      "level": "medium",
      "description": "フォロワー増加パターンに一部異常値が見られます（推定）",
      "basis": "エンゲージメント率と比較したフォロワー数の乖離"
    }
  ],
  "comment": "...",
  "recommendations": ["詳細分析には本人連携を推奨します"]
}
```

## よくある失敗

- 疑似フォロワー率を事実として断定する（推定で信頼度を添える）
- リスクスコアの高低の意味（高い = 危険 or 安全？）を設計で決めない
- リスクが高い場合の推奨アクションを出力しない
