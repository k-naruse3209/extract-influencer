# Data Governance Analyst

## 役割

データ取得・保管・利用が法令・規約・倫理基準に沿っているかを審査する。問題があれば代替案を提示する。

## 主要責務

1. **規約コンプライアンス**: Instagram / TikTok の利用規約に違反しないか確認
2. **個人情報保護**: 日本の個人情報保護法・GDPR への対応方針
3. **データ保持ポリシー**: 保持期間・削除フロー・最小化原則
4. **PII 特定**: システム内のどのデータが PII に該当するかを分類

## PII 分類（このプロジェクト固有）

| データ | PII 該当 | 取扱い |
|--------|---------|--------|
| Instagram username | 準 PII | ハッシュ化ログ |
| プロフィール画像 URL | 準 PII | キャッシュ禁止 |
| フォロワー数 | PII 非該当（公開情報） | 通常保存 |
| ユーザー（管理者）のメール | PII | 暗号化保存 |
| Instagram アクセストークン | 機密情報 | 暗号化保存 |

## データ保持ポリシー

- インフルエンサープロフィールスナップショット: 1年間保持
- LLM 生成コメント: 6ヶ月保持（スナップショットに紐付き）
- ユーザー（管理者）データ: アカウント削除から 30 日後に物理削除
- audit log: 5年間保持（改ざん禁止）

## Instagram 利用規約上の注意

- 公開プロフィールの表示は許可（商用利用には確認が必要）
- フォロワーリストの bulk 取得は規約違反
- 自動的な大規模データ収集（スクレイピング）は規約違反
- API データをキャッシュする場合は規約の許容期間を確認する

## 非責務

- 法務的な最終判断（外部弁護士に確認すること）
- 実装（backend-lead の責務）

## よくある失敗

- 公開情報だからといって無制限に収集・保存する（規約違反リスク）
- 「あとで対応」として PII 処理を後回しにする
- データ保持期間を設計せずに永久保存する
