# Data Source Compliance

## 目的

新しいデータソースを追加する前に、取得可否・規約遵守・PII 取扱い・法務リスクを評価する。

## 評価フロー

1. `templates/source-evaluation-template.md` を使ってソースを評価する
2. 評価結果を source-strategist に共有する
3. 重大な問題がある場合は data-governance-analyst にエスカレーション
4. 承認後に実装を開始する

## Instagram 固有の注意事項

### 取得可能（公式）

- 公開プロフィール基本情報
- 公開アカウントの投稿一覧
- いいね数・コメント数（公開投稿のみ）
- フォロワー数（公開アカウントのみ）

### 取得不可 / 要本人連携

- フォロワーリスト → **要 Business Account + 本人連携**
- インサイト（リーチ・インプレッション）→ **要本人連携**
- ストーリーズの詳細 → **要本人連携**
- DM → **取得禁止**

### 禁止事項

- IP ローテーションによるスクレイピング → **規約違反**
- 第三者データプロバイダー（非公認）の利用 → **要審査**
- フォロワーリストの bulk 取得 → **規約違反**
- レート制限の意図的な回避 → **規約違反**

## TikTok 固有の注意事項

### MVP での方針

- **大規模自動探索: 禁止**
- **本人連携のみ許可**（TikTok Login Kit）
- **手動入力を第一選択肢**とする

### 禁止事項

- TikTok Research API の商用利用 → **利用規約違反**
- 非公式 API の利用 → **規約違反・停止リスク**
- Headless Browser でのスクレイピング → **禁止**

## PII チェック

新しいデータを追加する際は以下を確認:
- このデータは PII（個人識別情報）か
- ログに出力してよいか
- 保持期間はどうするか
- 削除フローが設計されているか

## gotchas

- 「公開されているから取得できる」は規約的に誤りのことがある。API 利用規約を必ず確認する
- TikTok の API 仕様は頻繁に変更される。実装前に最新ドキュメントを確認する
- Instagram Business API のスコープは申請が必要。実装前に申請状況を確認する
