# E2E テストチェックリスト

## 実行前準備

- [ ] ステージング環境が起動しているか
- [ ] テスト用 DB がクリーンな状態か
- [ ] Instagram API サンドボックスが利用可能か
- [ ] テスト用アカウントのトークンが有効か

---

## フロー 1: 候補検索

- [ ] ジャンルフィルタを設定して検索できる
- [ ] フォロワー数レンジフィルタが反映される
- [ ] リスクレベルフィルタが反映される
- [ ] 検索結果0件のとき「候補が見つかりません」が表示される
- [ ] ローディング中にスケルトンが表示される
- [ ] 推定値フィールドに ※ マークが表示される
- [ ] ※ マークをホバーすると根拠ツールチップが表示される

---

## フロー 2: 候補保存

- [ ] 候補一覧から「保存」ボタンで保存できる
- [ ] 保存済み一覧に即時反映される
- [ ] 既に保存済みの候補は「保存済み」表示になる
- [ ] 保存上限（MVP: 100件）を超えたときエラーが表示される

---

## フロー 3: 比較

- [ ] 最大10件選択して比較画面に遷移できる
- [ ] 11件以上選択しようとすると警告が表示される
- [ ] 比較画面で各スコアが横並びで表示される
- [ ] 比較画面から PDF 出力できる
- [ ] 比較画面から CSV エクスポートできる

---

## フロー 4: PDF レポート生成

- [ ] PDF が生成されダウンロードできる
- [ ] PDF にアカウント名・フォロワー数・スコアが含まれている
- [ ] 推定値フィールドに「※推定値」の注記がある
- [ ] `status: "unavailable"` のフィールドに「-」が表示される（空白でない）
- [ ] LLM生成コメントに「AI生成」の注記がある

---

## フロー 5: CSV エクスポート

- [ ] CSV がダウンロードできる
- [ ] Excel で開いて文字化けしない（UTF-8 BOM 付き確認）
- [ ] 必須カラムが全て含まれている
- [ ] unavailable フィールドが「-」または「N/A」で出力される

---

## フロー 6: 管理画面

- [ ] admin ロールのみアクセスできる
- [ ] viewer ロールでアクセスしようとすると 403 が返る
- [ ] ユーザー追加・削除が audit log に記録される

---

## 完了判定

全チェックが通ったら e2e-runner が「E2E 全件グリーン」を記録し、release-manager に報告する。
